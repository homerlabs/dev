Algorithm 3- Mono Sector

Main Page

This algorithm can be performed on rows and columns. It compares the data in the row (or column) with data shared with a block. A row sector contains 3 consecutive columns that reside within the same block and a column sector contains 3 consecutive rows that reside with the same block. For example, rows 1, 2, and 3 each have 3 cells in block 1 (columns 1, 2, and 3), 3 cells in block 2 (columns 4, 5, and 6), and 3 cells in block 3 (columns 7, 8, and 9).

Notice on row 3 cell 8 has been reduced from '35678' to '3678'. The value '5' has been removed from the 3rd sector. This is because the algorthim noticed that the value '5' must reside somewhere on row 2's 3rd sector. Therefore the solution '5' cannot be on the 3rd sector for rows 1 or 3.

In row 4 the value '1' has been removed from the 1st sector. This is because on row 6 there is no '1' in it's 2nd or 3rd sector. Therefore the solution '1' must lie on sector 1 for row 6 which means that it cannot reside in sector 1 for rows 4 and 5.
