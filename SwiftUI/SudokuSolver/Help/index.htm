Sudoku Solver Help Guide

Initial Screen

When Sudoku Solver launches, it loads a Sudoku puzzle from "www.websudoku.com".
Above is a screenshot of a typical puzzle after it has been loaded, parsed, and then trimmed.

Trimming is a process where "Given" values are removed from the appropriate nodes in the rows, columns, and blocks of the puzzle.

The unsolved node count is calculated by counting all the numbers in all the cells then subtracting 81 as there will be 0 unsolved nodes when there is only 1 number in each cell. When the unsolved node count reaches 0 the puzzle is solved and the Solve button is disabled as no more solving can take place.

The puzzle is solved by repeated application of the three algorithms available through the Algorithm Selection bar just under the puzzle cell array. The following pages describe in detail how the algorithms work:

Algorithm 1: Mono Cell
Algorithm 2: Find Sets
Algorithm 3: Mono Sector

Special Features Available from the Menu

New (Command-N) Not really a special feature but wanted to include it in the menu descriptions. Works the same as hitting the 'New Puzzle' button.

Fetch (Shift-Command-N) This will request the puzzle id from the User and then fetch it. This was the original purpose of the app. To help me solve really hard puzzles that I was stuck on.

Save (Command-S) At anytime, a snapshot of the puzzle can be saved to a '.txt' file. Puzzle Data Format Description

Open (Command-O) A new puzzle can be loaded from a properly formatted text file. Typically this file would be created by a 'Save' operation but it could also be hand crafted.
