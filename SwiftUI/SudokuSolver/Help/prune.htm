Prune/Solve and Undo Buttons

Main Page

Prune

When a puzzle is imported from "web.sudoku.com" only the solved cells are known (the Given). Withoout any knowledge of adjacent cells the possible solutions for the 'Unsolved' cells are {1,2,3,4,5,6,7,8,9} otherwise known as 'the full set'. By 'pruning' however, many degrees of freedom can be eliminated. For any given cell, pruning can be applied to the Row, Column, and Block that the cell is located in. Pruning should be applied to all cells for all 3 modes (Rows, Columns, and Blocks) after initial puzzle import. Aso, after every 'Solve' operation the data set is pruned post Solve operation. Specifically, any time any cell becomes 'Solved', the data set should be pruned.

Solve

After the puzzle has been imported and pruned the Prune button becomes the Solve button. By repeated application of all 3 solve algorithms the puzzle will eventually reach zero Unsolved Nodes. At this time the puzzle has become solved and the Solve button becomes disabled because no further solve operations are possible.

Undo

A copy of the previous data set is maintained. If the Undo button is tapped this previous data set is copied to the current data set. Nested Undos are not permitted so after each Undo operation the Undo button becomes disabled. It becomes re-enabled after the next Solve operation.
